---

Cada sitio de Mintlify necesita un archivo `docs.json` con la configuración básica. Aprende más sobre las [propiedades](#properties) a continuación.

## Propiedades

<ResponseField name="name" type="string" required>
Nombre de tu proyecto. Se utiliza para el título global.

Ejemplo: `mintlify`

</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  Un array de grupos con todas las páginas dentro de ese grupo
  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
    El nombre del grupo.

    Ejemplo: `Settings`

    </ResponseField>
    <ResponseField name="pages" type="string[]">
    Las rutas relativas a los archivos markdown que servirán como páginas.

    Ejemplo: `["customization", "page"]`

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  Ruta a la imagen del logo u objeto con ruta a las imágenes del logo para modo "light" y "dark"
  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      Ruta al logo en modo claro
    </ResponseField>
    <ResponseField name="dark" type="string">
      Ruta al logo en modo oscuro
    </ResponseField>
    <ResponseField name="href" type="string" default="/">
      A dónde te lleva hacer clic en el logo
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  Ruta a la imagen del favicon
</ResponseField>

<ResponseField name="colors" type="Colors">
  Códigos de color hexadecimales para tu tema global
  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      El color primario. Se utiliza más frecuentemente para contenido destacado, encabezados de sección, acentos, en modo claro
    </ResponseField>
    <ResponseField name="light" type="string">
      El color primario para modo oscuro. Se utiliza más frecuentemente para contenido destacado, encabezados de sección, acentos, en modo oscuro
    </ResponseField>
    <ResponseField name="dark" type="string">
      El color primario para botones importantes
    </ResponseField>
    <ResponseField name="background" type="object">
      El color del fondo tanto en modo claro como oscuro
      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          El código de color hexadecimal del fondo en modo claro
        </ResponseField>
        <ResponseField name="dark" type="string" required>
          El código de color hexadecimal del fondo en modo oscuro
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  Array de `name`s y `url`s de enlaces que quieres incluir en la barra superior
  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
    El nombre del botón.

    Ejemplo: `Contact us`
    </ResponseField>
    <ResponseField name="url" type="string">
    La url una vez que haces clic en el botón. Ejemplo: `https://mintlify.com/docs`
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
    Link muestra un botón. GitHub muestra la información del repositorio en la url proporcionada incluyendo el número de estrellas de GitHub.
    </ResponseField>
    <ResponseField name="url" type="string">
    Si `link`: A qué enlaza el botón.
    
    Si `github`: Enlace al repositorio para cargar información de GitHub.
    </ResponseField>
    <ResponseField name="name" type="string">
    Texto dentro del botón. Solo requerido si `type` es un `link`.
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  Array de nombres de versiones. Solo usa esto si quieres mostrar diferentes versiones
  de documentación con un menú desplegable en la barra de navegación.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  Un array de los anclajes, incluye el `icon`, `color`, y `url`.
  <Expandable title="Anchor">
    <ResponseField name="icon" type="string">
    El ícono de [Font Awesome](https://fontawesome.com/search?q=heart) utilizado para destacar el anclaje.

    Ejemplo: `comments`
    </ResponseField>
    <ResponseField name="name" type="string">
    El nombre de la etiqueta del anclaje.

    Ejemplo: `Community`
    </ResponseField>
    <ResponseField name="url" type="string">
      El inicio de la URL que marca qué páginas van en el anclaje. Generalmente, este es el nombre de la carpeta donde colocas tus páginas.
    </ResponseField>
    <ResponseField name="color" type="string">
      El color hexadecimal del fondo del ícono del anclaje. También puede ser un degradado si pasas un objeto con las propiedades `from` y `to` que son cada una un color hexadecimal.
    </ResponseField>
    <ResponseField name="version" type="string">
      Se utiliza si quieres ocultar un anclaje hasta que se seleccione la versión correcta de la documentación.
    </ResponseField>
    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      Pasa `true` si quieres ocultar el anclaje hasta que enlaces directamente a alguien a la documentación dentro de él.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      Uno de: "brands", "duotone", "light", "sharp-solid", "solid", o "thin"
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  Anula las configuraciones predeterminadas para el anclaje superior.
  <Expandable title="Object">
    <ResponseField name="name" default="Documentation" type="string">
      El nombre del anclaje superior
    </ResponseField>
    <ResponseField name="icon" default="book-open" type="string">
      Ícono de Font Awesome.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      Uno de: "brands", "duotone", "light", "sharp-solid", "solid", o "thin"
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  Un array de pestañas de navegación.
  <Expandable title="Tabs">
    <ResponseField name="name" type="string">
      El nombre de la etiqueta de la pestaña.
    </ResponseField>
    <ResponseField name="url" type="string">
      El inicio de la URL que marca qué páginas van en la pestaña. Generalmente, este
      es el nombre de la carpeta donde colocas tus páginas.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  Configuración para ajustes de API. Aprende más sobre páginas de API en [Componentes de API](/en/api-playground/demo).
  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      La URL base para todos los endpoints de API. Si `baseUrl` es un array, habilitará múltiples opciones de URL base que el usuario puede alternar.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type='"bearer" | "basic" | "key"'>
          La estrategia de autenticación utilizada para todos los endpoints de API.
        </ResponseField>
        <ResponseField name="name" type="string">
        El nombre del parámetro de autenticación utilizado en el playground de API.

        Si el método es `basic`, el formato debe ser `[usernameName]:[passwordName]`
        </ResponseField>
        <ResponseField name="inputPrefix" type="string">
        El valor predeterminado que está diseñado para ser un prefijo para el campo de entrada de autenticación.

        Por ejemplo, si un `inputPrefix` de `AuthKey` heredaría el resultado de entrada predeterminado del campo de autenticación como `AuthKey`.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      Configuraciones para el playground de API

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type='"show" | "simple" | "hide"'>
          Si el playground se está mostrando, oculto, o solo mostrando el endpoint sin interactividad adicional del usuario `simple`

          Aprende más en las [guías del playground](/en/api-playground/demo)
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      Habilitar esta bandera asegura que el orden de las claves en las páginas OpenAPI coincida con el orden de claves definido en el archivo OpenAPI.

      <Warning>Este comportamiento pronto será habilitado por defecto, momento en el cual este campo será deprecado.</Warning>
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  Una cadena o un array de cadenas de URL(s) o ruta(s) relativa(s) que apuntan a tu archivo OpenAPI.
  
  Ejemplos:
  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```
    ```json Relative
    "openapi": "/openapi.json"
    ```
    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>

</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  Un objeto de cuentas de redes sociales donde el par clave:propiedad representa la plataforma de redes sociales y la URL de la cuenta.
  
  Ejemplo: 
  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```
  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
    Uno de los siguientes valores `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`
    
    Ejemplo: `x`
    </ResponseField>
    <ResponseField name="property" type="string">
    La URL a la plataforma social.
    
    Ejemplo: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  Configuraciones para habilitar botones de retroalimentación

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
    Habilita un botón para permitir a los usuarios sugerir ediciones a través de pull requests
    </ResponseField>
    <ResponseField name="raiseIssue" type="boolean" default="false">
    Habilita un botón para permitir a los usuarios reportar un problema sobre la documentación
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  Personaliza el botón de alternancia del modo oscuro.
  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      Establece si siempre quieres mostrar el modo claro u oscuro para nuevos usuarios. Cuando no se establece, por defecto usamos el mismo modo que el sistema operativo del usuario.
    </ResponseField>
    <ResponseField name="isHidden" type="boolean" default="false">
      Establece en true para ocultar el botón de alternancia del modo oscuro/claro. Puedes combinar `isHidden` con `default` para forzar que tu documentación solo use el modo claro u oscuro. Por ejemplo:
      
      <CodeGroup>
      ```json Solo Modo Oscuro
      "modeToggle": {
        "default": "dark",
        "isHidden": true
      }
      ```

      ```json Solo Modo Claro
      "modeToggle": {
        "default": "light",
        "isHidden": true
      }
      ```
      </CodeGroup>

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  Una imagen de fondo que se mostrará detrás de cada página. Ve el ejemplo con
  [Infisical](https://infisical.com/docs) y [FRPC](https://frpc.io).
</ResponseField>
